The workflow

Speed first. Depth second. Evidence always.

Each stage has a different job. Separating them prevents one model's first answer from becoming an unquestioned implementation plan.

01

Fast diagnosis

Start with Claude Opus 4.8

I first asked Opus to inspect the reported behaviour, trace the code path, and produce a structured diagnosis. Claude models are effective at turning a broad problem into a useful first draft quickly, so the output became the initial ISSUE.md—a concrete artefact that another reviewer could challenge.

Output artefact
ISSUE.md
Symptoms → root causes → proposed fixes → validation
02

Independent challenge

Use GPT-5.6-sol as the second set of eyes

Next, I asked GPT to audit the draft rather than repeat the investigation. GPT models tend to spend more time on edge cases and coverage. The second pass found blind spots in the executable path, corrected overstated evidence, and separated must-fix defects from follow-up concerns. Different model families are useful precisely because their failure modes are different.

03

Task orchestration

Freeze the contract, then parallelize

Once the diagnosis was stable, I translated it into AGENT-TASK.md. The plan partitions ownership by layer, gives each agent disjoint files, and freezes the interfaces before work begins. That lets multiple subagents execute at the same time without editing collisions—higher throughput without losing integration discipline. On a larger project I would use Pencil or a dedicated task system; for a focused code test, Markdown keeps the process transparent.

04

Human quality gate

Trust the result only after it survives review

The last gate stays human. I manually test the reported behaviours, compare the output against database ground truth, and review the final diff. I look especially for duplicated logic, broken contracts, unnecessary abstractions, and scope creep. AI has no hesitation about rebuilding a working system, so I explicitly enforce DRY, minimal changes, and the assignment's “debug—do not rebuild” constraint.

The takeaway

AI accelerates the work. Engineering judgment controls the outcome.

The process is deliberately layered: one model creates momentum, another challenges assumptions, agents increase throughput, and a human-owned quality gate decides when the result is actually ready.
